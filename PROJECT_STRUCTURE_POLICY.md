# 项目目录与版本政策

> 版本：v1.0  
> 生效日期：2026-07-14  
> 适用根目录：`D:\MMSA-CH-SIMS`

本项目是既有自定义 Python 研究仓库。历史脚本、报告和结果保持原位；本政策不授权移动、删除、覆盖或批量重命名现有资产。

| 区域 | 规范位置 | Git策略 | 说明 |
|---|---|---|---|
| 原始数据 | `data/raw/` | 不提交实体数据 | 下载后只读保存；来源、许可、大小和SHA-256写入台账 |
| 处理数据 | `data/processed/` | 不提交实体数据 | 只能由版本化脚本和清单生成 |
| 数据清单 | `data/manifests/` | 提交小型清单 | 不含评论正文、用户标识或秘密下载链接 |
| 新主线代码 | `src/` | 提交 | 历史根目录脚本暂不移动；新模块逐步放入此处 |
| 实验配置 | `configs/` | 提交 | 正式运行必须绑定单一配置文件 |
| 测试 | `tests/` | 提交 | 泄漏、配置、数据划分和评测测试 |
| 实验登记 | `experiments/` | 提交登记与小型摘要 | 大型中间结果、缓存和权重不得提交 |
| 运行结果 | `results/`、`logs/` | 默认不提交 | 只把经审查的论文表格/摘要复制到文稿证据区 |
| 模型与索引 | `models/`、`saved_models/`、`indices/` | 不提交 | 另行决定对象存储或发布仓库 |
| 文稿 | `paper/` | 提交 | 正文、附录、图表源和投稿材料 |
| 文献 | `references/` | 提交 | BibTeX、检索日志和引用审计，不存付费全文 |

原始评论、平台用户标识、Cookie、API key、模型权重和大型数据不得进入Git。结构合规不等于数据质量、实验可复现性、统计有效性或论文质量已经通过。
