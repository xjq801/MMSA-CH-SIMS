## 1. 实验目的

验证自建数据集是否适合引入 GNN / Temporal GNN 建模群体情绪传播拓扑。

本次先做最小可跑版本：

```text
视频 = 节点
原始48维统计特征 = 节点特征
发布者时间相邻关系 = temporal edge
共享评论用户关系 = user co-comment edge
↓
MLP baseline vs GCN
↓
极端群体情绪二分类
```

没有安装 PyG/DGL，直接用 PyTorch 实现两层 GCN。

## 3. MLP vs GCN 结果

实验设置：

- 80/20 随机分层测试划分；
- 训练集中再切验证集；
- seeds：1111、1112、1113；
- 表中为均值 ± 标准差，单位为 %。

| 模型 / 图结构 | Accuracy | Precision | Recall | F1 Score |
|---|---:|---:|---:|---:|
| MLP，仅48维，无图 | 71.46 ± 0.37 | 74.27 ± 0.76 | 62.44 ± 0.18 | 67.84 ± 0.24 |
| GCN，发布者时间相邻边 | **73.44 ± 0.86** | 76.62 ± 1.69 | **64.68 ± 0.77** | **70.13 ± 0.74** |
| GCN，共享评论用户边 | 67.45 ± 0.44 | 75.94 ± 0.46 | 47.51 ± 0.98 | 58.45 ± 0.81 |
| GCN，时间边 + 用户边 | 68.76 ± 1.28 | **79.02 ± 0.62** | 47.89 ± 2.96 | 59.59 ± 2.46 |

## 4. 结论

### 4.1 Temporal GNN 方向有信号

只使用“发布者时间相邻边”时，GCN 相比 MLP 有提升：

- Accuracy：71.46% → 73.44%
- F1：67.84% → 70.13%

这说明视频之间的发布顺序/时间相邻传播结构确实包含有效信息。

### 4.2 共享评论用户边当前不适合直接使用

共享评论用户边太稠密，GCN 结果明显下降：

- F1 从 MLP 的 67.84% 降到 58.45%；
- 合并所有边后 F1 也只有 59.59%。

这说明不能简单地把“评论过同一视频的用户”全部连边。这样会形成过密图，导致不同话题、不同情绪的视频表示被过度平均。

### 4.3 后续应该优先做 Temporal GNN，而不是普通全图 GNN

当前结果支持：

> 群体情绪传播更适合先用发布者时间序列 / 视频发布时间邻接建模，而不是直接用高密度用户共现图。

## 5. 下一步建议

推荐继续做：

1. Temporal GNN / Temporal GraphSAGE：
   - 节点：视频；
   - 边：同发布者相邻视频、同话题时间邻近视频；
   - 节点特征：48维 + LLM情绪元；
   - 目标：预测极端群体情绪。

2. 用户共现边不要直接全量使用，应该加筛选：
   - 只保留高活跃且高可信用户；
   - 按共同评论数量设置阈值；
   - 按时间窗口保留近邻；
   - 或把用户信息做成节点特征，而不是直接连密集边。

3. 论文表述建议：

> 初步传播拓扑实验表明，基于发布者时间相邻关系构建的时序图能够提升轻量深度模型的预测性能，而未经筛选的用户共评图由于边密度过高，可能导致节点表示过平滑。该结果说明群体情绪建模应更关注时间演化传播结构，而非简单用户共现关系。

## 6. 当前是否作为主结果？

不建议现在作为最终主结果。它更适合作为“传播拓扑可行性实验”或“下一阶段方法升级依据”。

原因：

- GCN 仍低于 CatBoost；
- 当前 GCN 只用了48维特征，没有融合 LLM 情绪元；
- 目前是随机划分，还没有做发布者分组划分；
- 用户共现边需要进一步筛选。

下一版应做：

```text
48维传播统计特征
+ StepFun LLM 情绪元
+ 发布者时间相邻图
↓
Temporal GCN / GraphSAGE
```
