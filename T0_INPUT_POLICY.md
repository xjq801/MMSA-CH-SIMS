# T0 输入与泄漏防护政策

> 版本：v1.0  
> 生效日期：2026-07-14  
> 上位纲要：`TAFFC_CH4_10_MONTH_MASTER_PLAN_20260713.md` v1.5  
> 适用范围：所有数据处理、检索建库、训练、验证、测试和论文结果。

## 1. 任务定义

T0 主任务是在目标视频或帖子发布时，预测其随后受众评论形成的群体情绪分布、群体分歧、风险与预测不确定性。

T0 不等于“观察发布后的评论再判断情绪”，也不等于传播过程建模。T+Δ 早期预警只有在评论时间戳和观察窗口可靠时才可作为独立次任务，且必须使用独立配置、数据列和结果表。

## 2. 统计与划分单位

- 统计单位：视频或帖子。
- 数据划分单位：视频或帖子；评论不得作为独立样本跨越 train/dev/test。
- bootstrap 与显著性分析单位：视频或帖子，不把 seed、fold 或评论数当独立样本量。
- 标签聚合单位：同一视频或帖子在预先冻结的响应窗口内形成的经验受众情绪分布。

## 3. T0 允许输入

仅允许使用在目标内容发布时真实可获得、且不由未来响应产生的信息：

- 标题、正文、简介、标签和话题；
- 封面、发布时已有的图像或视频内容及其离线特征；
- 发布者在目标发布时已公开的静态或历史元数据；
- 明确记录为发布时可见、且不依赖未来互动的内容属性；
- 训练集或合规历史案例的内容表示、历史受众反应分布、置信度、域信息与时间信息；
- 模态存在/缺失掩码和发布时即可计算的模态质量指标。

所有输入字段必须在数据字典中登记 `available_at_t0=true` 及其来源。没有证据证明 T0 可用的字段，默认禁止。

## 4. T0 禁止输入

- 目标视频或帖子的任何未来评论、评论情绪、点赞评论、弹幕响应或评论聚合特征；
- 最终或未来播放量、点赞量、转发量、收藏量、评论量及其增长曲线；
- 发布后推荐结果、相关推荐列表或由未来行为更新的热度/排名；
- 标签生成过程中使用过的任何目标响应特征；
- test 样本、未来样本或其派生表示进入检索库、原型库、归一化统计或特征选择；
- 利用全量数据预先构图后产生的跨 split 图特征或邻居信息；
- 从文件顺序推断的伪时间，以及无法证明早于目标时点的数据；
- 人工查看 test 标签后设置的阈值、主指标、超参数或停止条件。

## 5. 评论特权监督边界

- train 评论可以供反应教师学习标签、个体反应和视频级分布。
- dev 评论只能用于预先声明的模型选择与校准，不能进入训练记忆。
- test 评论只用于最终标签构建和盲评，必须与模型输入、检索索引和调参流程物理隔离。
- 内容学生在训练和推理时均只接受 T0 允许输入；教师知识只能通过预先定义的蒸馏目标传递。
- 若使用自动评论情绪模型生成银标，银标不得被描述为独立人工测试真值。

## 6. 检索索引生命周期

1. 先按视频/帖子完成 train/dev/test 划分，再建立任何索引。
2. 主实验索引只包含 train 样本；dev/test 样本不得写入索引。
3. 有可靠时间戳时，候选还必须满足 `candidate_publish_time < query_publish_time`。
4. 无可靠时间戳时，只允许声称 train-only 检索，不得声称严格历史预测或时间泛化。
5. 每个索引必须记录数据版本、split 版本、构建配置、样本 ID 清单、创建时间和 SHA-256。
6. 每次评测前运行断言：查询 ID 不在索引中、候选不来自 dev/test、同源近重复不跨 split。

## 7. 标签窗口与阈值

- 每个数据集的响应窗口必须在查看 test 结果前写入数据清单；不同数据集可以不同，但不得在同一结果表中隐式混用。
- 主输出是 K 类经验情绪分布；群体分歧优先用分布熵或离散度。
- 二分类风险仅作兼容次任务；阈值只在 train 或预先授权的 dev 数据上冻结。
- 自建 CUC-IGPE-v2 没有独立人工真值时，仅作银标签/无标签压力测试，不报告其准确率、F1 或校准提升为核心证据。

## 8. 最低验收测试

每个正式实验必须通过以下检查：

- train/dev/test 的视频或帖子 ID 交集为空；
- 同一视频的评论不跨 split；
- 目标评论、最终互动量和未来推荐字段不在模型输入列中；
- 检索索引样本集合是 train 的子集；
- 时间协议启用时，所有候选严格早于查询；
- 归一化、特征选择、聚类、原型和阈值只在允许数据上拟合；
- 配置明确记录 `task_timepoint=T0`、数据版本、split 版本、标签窗口和输入模态；
- 预测文件可反查到配置、模型、索引和数据清单。

任一检查失败时，该次运行标记为 `LEAKAGE_BLOCKED`，结果不得进入论文表格或主结论。

## 9. 变更纪律

本政策若需修改，必须记录版本、理由、受影响实验和需要重跑的结果。不得为了改善 test 表现而追溯修改 T0 输入、标签窗口、阈值或划分。
