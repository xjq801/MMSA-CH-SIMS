# 面向 IEEE Transactions on Affective Computing 的差距评估与实施路线

日期：2026-07-13

## 一、总体判断

研究主题与 IEEE Transactions on Affective Computing（TAC）的范围匹配：TAC 官方范围包含多模态情感识别、群体情绪识别、情感语料采集与标注。但当前工作仍是“毕业论文原型 + 探索性实验”，距离 TAC 可投稿版本有实质差距，当前准备度约为 30%–40%。这一比例不是录用概率，而是按问题、方法、数据、实验和可复现性五个方面估计的工作完成度。

核心问题不是 CatBoost 比深度学习高，而是当前自建数据只有人工表格特征，随机划分中容易学习数据集局部规律。正式实验已显示：CatBoost 在随机划分上很高，但时间划分和发布者分组划分明显下降，说明泛化而非单次准确率才是更重要的科学问题。

## 二、主要差距

| 方面 | 当前状态 | TAC 需要补齐的证据 |
|---|---|---|
| 研究问题 | 主要追求预测准确率 | 明确研究“不完整模态、跨域迁移、群体情绪传播”中的一个核心问题 |
| 多模态数据 | 自建数据缺少音频和视频 | 在至少两个完整公开 benchmark 上训练和验证 |
| 方法创新 | Self-MM + 权重 + 原型 + Temporal，模块较多 | 形成一个统一机制，并说明为什么有效、在何时失效 |
| 标签有效性 | 二分类极端情绪，部分标签与连续值存在冲突 | 情感构念定义、标注规范、一致性和阈值敏感性分析 |
| 泛化 | 随机划分高，时间/发布者划分下降 | speaker/topic/time/domain held-out 和跨数据集实验 |
| Baseline | Self-MM、CatBoost、HGB 等 | benchmark 官方强基线 + 近年缺失模态方法，统一特征和预算 |
| 统计与稳健性 | 小样本、少量种子 | 5 个种子、置信区间、显著性、完整缺失模式、校准与效率 |
| 可复现性 | 已有代码和环境修复记录 | 固定配置、数据 lineage、公开代码、日志、模型权重和伦理说明 |

## 三、建议冻结的论文主问题

建议把论文主问题冻结为：

> 面向自然缺失模态和跨域变化的可靠多模态情感建模：在完整公开多模态 benchmark 上学习可迁移的情感表示，在缺失音频/视觉的社交媒体数据上进行可靠推断，并进一步建模群体情绪的时间演化。

这条路线保留原论文“群体情绪”的主题，也合理利用自建数据的缺失模态现实。

不建议把贡献写成“把 CatBoost 换成 Transformer”或“加入 LLM 后准确率提高”。这两种表述不足以构成 TAC 级别的科学贡献。

## 四、数据布局

### 主 benchmark

1. CH-SIMS v2：中文、多模态、具有独立单模态标签，适合研究音频和视觉是否真正提供增量。
2. CMU-MOSEI：规模更大、说话人更多，适合验证跨说话人泛化和英文数据迁移。
3. 可选 MERBench/MER2023：用于中文多模态情绪识别和统一评测，增强任务覆盖。

### 自建数据的重新定位

- 不再把自建数据称为完整“音频-视觉-文本多模态数据集”。
- 定位为“文本 + 传播结构 + 交互元数据的群体情绪动态外部验证集”。
- 若原视频 URL 仍可合法访问，可重新下载并恢复音视频；若无法恢复，明确记录为自然缺失模态。
- 不用 LLM 生成伪音频或伪视频冒充真实模态。
- 补做标签审计：标注规范、双人/三人标注、一致性、连续极化分数及二分类阈值敏感性。

## 五、推荐模型结构

建议模型暂命名为 Reliability-aware Prototype Distillation Network（名称后续需查新）：

1. 三个编码器：文本、音频、视觉。
2. 可靠性估计器：根据模态质量、缺失状态和预测不确定性生成动态权重。
3. 情绪原型库：学习多情绪/多强度原型，不只使用正负二原型。
4. 完整模态教师—缺失模态学生：完整 benchmark 上训练教师，通过蒸馏、对比学习或原型传输帮助缺失模态学生。
5. Temporal/Graph 下游头：只在社交媒体群体情绪数据上使用，建模时间和传播结构，不与 utterance-level benchmark 主任务混在一起。

核心创新必须落在“可靠性估计 + 原型传输/蒸馏如何解决自然缺失和域变化”，而不是五个模块简单相加。

## 六、必须完成的实验矩阵

### Benchmark 主实验

- 完整 T+A+V。
- 单模态 T/A/V。
- 缺失一个模态：T+A、T+V、A+V。
- 缺失两个模态。
- 随机缺失率 10%、30%、50%、70%。
- speaker/topic held-out。
- CH-SIMS v2 与 CMU-MOSEI 跨数据集迁移。

### Baseline

- 传统模型：CatBoost、HGB，仅用于表格特征基线。
- 多模态基础模型：Self-MM 和 benchmark 官方实现。
- 缺失模态基线：至少选择 2–3 个公开代码、同任务的近期方法。
- LLM 辅助特征必须单独作为消融，不应替代人工测试标签。

### 评价

- 分类：Accuracy、Macro-F1、Weighted-F1、每类 Recall、混淆矩阵。
- 回归：MAE、Corr。
- 可靠性：ECE/Brier、置信度—错误关系。
- 泛化：跨说话人、跨主题、跨时间、跨数据集。
- 工程：参数量、FLOPs、推理时间、显存。
- 统计：至少 5 个种子、bootstrap 95% CI、配对检验和多重比较校正。

### 消融

- 去可靠性门控。
- 去情绪原型。
- 去教师蒸馏。
- 去缺失模态训练。
- 去 LLM 情绪元。
- 去 Temporal/Graph。
- 固定权重与自适应权重比较。

## 七、分阶段实施路线

### Phase 0：问题和数据冻结（2–4 周）

- 冻结主问题、标签定义和 primary metric。
- 完成 CH-SIMS v2、CMU-MOSEI/MERBench 许可和数据可用性检查。
- 明确自建数据哪些视频可恢复、哪些属于自然缺失。

退出条件：数据、划分、指标和失败条件全部写入实验计划。

### Phase 1：强基线复现（4–8 周）

- 在统一框架跑通 Self-MM、CatBoost/HGB 和 2–3 个近期缺失模态 baseline。
- 先复现论文或官方仓库允许误差范围内的结果，再开发新模型。

退出条件：所有 baseline 使用同一特征、划分、调参预算和评测脚本。

### Phase 2：最小新模型（6–10 周）

- 先只实现可靠性估计 + 原型蒸馏两个核心模块。
- 在 CH-SIMS v2 上完成完整/缺失模态实验。
- 如果对强 baseline 没有稳定增益，停止增加模块，回到机制设计。

退出条件：至少两个数据集上取得稳定增益，或在鲁棒性/校准上有明确优势。

### Phase 3：泛化和群体情绪扩展（6–10 周）

- 增加 CMU-MOSEI 或 MERBench。
- 完成 speaker/topic/time/domain held-out。
- 将缺失模态学生迁移到自建群体情绪数据，再增加 Temporal/Graph 传播头。

退出条件：自建数据作为外部验证，而不是支撑全部结论的唯一数据。

### Phase 4：TAC 级实验和论文（6–8 周）

- 完整消融、失败案例、可解释性、统计、效率、公平性与伦理。
- 公开代码、配置、日志和可合法发布的数据索引。
- 预演 TAC 审稿：创新是否只是模块堆叠、缺失模态是否人工模拟、跨域是否成立、标签是否可信。

## 八、Go/No-Go 门槛

满足以下条件再以 TAC 为直接投稿目标：

1. 至少两个公开 benchmark。
2. 至少一个自然缺失或真实跨域设置，而非只有人工随机遮蔽。
3. 对强 baseline 的提升在多数设置和多个种子上成立，或在校准、鲁棒性、效率上形成清晰 Pareto 优势。
4. 自建标签有可审计的构念和一致性证据。
5. 跨说话人/主题/时间结果不出现灾难性下降，或论文能解释并针对性解决。
6. 代码、数据处理、划分和统计完全可复现。

若只完成 CH-SIMS 上的小幅模型改进、自建数据随机划分高分和少量消融，建议先投更匹配的中等期刊，不直接冲 TAC。

## 九、当前最优先的三件事

1. 下载并审计 CH-SIMS v2，确认音频、视频、文本和官方划分全部可用。
2. 复现一个完整模态 baseline 和一个缺失模态 baseline，建立统一评测脚本。
3. 冻结“可靠性估计 + 原型蒸馏”的最小模型，不再继续无目的叠加模块。
