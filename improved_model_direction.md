# 后续改进模型设计思路

## 1. 改进目标

结合前述 benchmark 实验结果可以发现，论文原有的 CatBoost 建模思路在粗粒度情绪极性判断上具有一定优势，但在细粒度情绪强度回归、多类别区分以及复杂跨模态交互建模方面仍存在明显不足。因此，后续工作若希望形成更具发表潜力的一作论文，研究重点不宜继续停留在传统特征工程与浅层集成学习层面，而应转向“公开 benchmark + 深度多模态建模”的技术路线。

具体而言，后续模型设计的核心目标可以概括为三点：一是提升对文本、音频、视觉三模态时序信息的建模能力；二是增强跨模态交互与互补信息融合能力；三是在保留原论文“群体情绪预测”研究特色的基础上，引入可解释的辅助机制，使模型既具备 benchmark 上的竞争力，又保留一定的方法辨识度。

## 2. 总体思路

下一阶段较为合理的方案，不是直接推翻原论文工作，而是在公开数据集 baseline 的基础上，构建一个“以 Self-MM 类方法为底座、吸收原论文思想的改进型多模态情绪预测模型”。其中，Self-MM 可以作为可复现、可比较的基础框架；而原论文可以贡献的内容，则应更多体现在“如何设计新的跨模态增强模块、情绪相关辅助特征、或带有可解释性的引导机制”上。

换句话说，下一篇文章的逻辑最好从“我有一个自建数据集和一套特征”转变为“我在公开基准上发现现有方法的不足，并提出一种新的多模态增强机制来解决该问题”。这种写法更符合期刊和学位论文答辩中对研究可信度、可重复性和创新表达的要求。

## 3. 可行的模型架构方案

一个比较适合你当前基础、同时又具备论文可写性的方案，是设计一个“主干网络 + 跨模态增强模块 + 情绪引导模块 + 回归/分类联合输出头”的结构。其基本流程如下：

1. 文本模态输入 BERT 或中文 RoBERTa 编码后的序列表示。
2. 音频模态输入预提取声学特征序列。
3. 视觉模态输入预提取视觉特征序列。
4. 分别使用轻量时序编码器对三种模态做单模态表示学习。
5. 引入跨模态交互模块，学习文本-音频、文本-视觉、音频-视觉之间的关联信息。
6. 设计情绪引导或置信度重加权模块，对不同模态的重要性进行自适应调整。
7. 最后通过融合层输出情绪预测结果，可同时支持回归与分类任务。

如果用更直白的话来讲，这个模型不是简单把三种模态拼起来，而是先分别学“每个模态自己在表达什么”，再学“它们彼此之间怎么互相补充”，最后再决定“当前这个样本里哪一种模态更值得相信”。

## 4. 你最适合做的创新点

对你现在这个课题来说，最推荐的不是发明一个特别庞大、特别复杂的新网络，而是在一个成熟 baseline 上做“一个清楚、能讲明白、能验证有效”的模块创新。比较适合你的创新方向有下面三类。

### 4.1 模态自适应权重机制

不同样本中三种模态的信息质量并不一致。有些视频中文本情绪信号最强，有些视频则更多依赖语音语调或面部表情。因此，可以设计一个模态权重分配模块，根据当前样本的特征质量、自身置信度或模态间一致性，动态调整文本、音频、视觉三种模态在最终融合中的贡献。

这个方向的优点是：

- 思路直观，容易解释；
- 和原论文“关注群体情绪判别有效信号”的思路有延续性；
- 实验上容易通过消融实验验证有效性；
- 写论文时容易形成“为什么这样做”和“这样做带来了什么提升”的完整闭环。

### 4.2 情绪原型或情绪先验引导模块

你可以把原论文中“情绪传播/群体反应具有某种结构性规律”的想法，转化成一种更适合 benchmark 的情绪引导机制。例如，在特征空间中学习若干情绪原型向量，分别代表典型正向、负向和中性状态，让样本在融合过程中主动向这些情绪原型对齐，或者以原型相似度作为辅助监督信号。

这个方向的好处是：

- 比单纯堆叠网络层更有“方法味道”；
- 更容易形成自己的命名和方法标签；
- 能够自然写出“模型可解释性”部分；
- 容易与普通融合模型拉开叙述差异。

### 4.3 跨模态一致性约束

在多模态情绪任务中，文本、语音和视觉并不总是一致的，但它们通常不会完全无关。因此，可以增加一种模态一致性约束，让模型在融合时既保留互补性，又避免不同模态表示完全割裂。比如，可以对共享情绪空间中的模态表示施加距离约束、对比学习损失，或者加入教师引导机制，促进不同模态在情绪判别层面形成更稳定的对齐关系。

这个方向通常比较受审稿人欢迎，因为它很符合多模态任务的核心问题，也更接近当前主流研究语言。

## 5. 推荐你采用的具体方案

如果从“工作量、创新性、可解释性、投稿成功率”四个角度一起考虑，我最建议你的版本是：

“以 Self-MM 为基础主干，引入模态自适应权重机制，并结合情绪原型引导，构建一个轻量级的多模态情绪增强模型。”

这个方案有几个现实优势。

第一，它不是完全另起炉灶，而是在已经跑通的公开 baseline 上做改进，实验更容易落地。

第二，它既有深度学习模型的结构创新，又保留了你原论文强调“情绪识别要抓有效信号”的研究味道，不会显得和学姐原工作完全脱节。

第三，它的创新点数量适中，不会因为堆得太多而让实验难以收敛、论文逻辑变乱。

第四，它很适合写消融实验。你可以自然设置：

- baseline：Self-MM
- baseline + 模态权重模块
- baseline + 情绪原型模块
- baseline + 两者结合

这样论文结构会非常清楚，审稿人也容易接受。

## 6. 可以怎么命名

论文的方法名不需要太花，但最好体现“多模态、自适应、情绪引导”三个关键词。下面这几个命名方向都可以参考：

- Adaptive Emotion-Guided Multimodal Network（AEGMN）
- Modality-Adaptive Emotion Fusion Network（MAEF-Net）
- Emotion Prototype Guided Multimodal Model（EPGM）
- Adaptive Cross-modal Emotion Enhancement Network（ACEE-Net）

如果你想保留“群体情绪预测”这个味道，也可以把题目写得偏应用一点，例如：

“基于情绪引导与模态自适应融合的多模态情绪预测方法”

这样的名字比较稳，不会太虚，也方便中文论文和投稿题目统一。

## 7. 后续实验设计建议

为了让后续论文更完整，建议实验部分至少包含以下内容：

1. 在 CH-SIMS 上与 Self-MM、TFN、LMF、MISA 等公开方法比较。
2. 报告二分类、多分类和回归多种指标，而不只看单一准确率。
3. 进行消融实验，验证新增模块的真实贡献。
4. 做可视化分析，例如不同模态权重的变化、情绪原型分布、典型样本预测案例。
5. 如条件允许，再增加一个中文 benchmark 进行跨数据集验证，以增强说服力。

## 8. 一句话建议

如果你的目标是“尽快发一个低级别期刊”，那么在现有工作上补 benchmark、补 baseline、补实验分析，已经可以开始尝试整理稿件。

如果你的目标是“争取学生一作，并且让文章更像一篇真正的多模态情绪计算论文”，那更推荐你走这条路线：先以 Self-MM 为起点复现 benchmark，再在其上加入一个清晰可验证的改进模块，把论文重心从“自建特征工程”转向“公开基准上的模型创新”。
