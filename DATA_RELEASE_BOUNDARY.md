# dataset-v1可发布/不可发布边界

## 当前判定

dataset-v1状态为`LOCAL_CANDIDATE_G1_BLOCKED`。本表描述“技术上可进入版本控制的最小证据包”，不构成公开发布授权；实际公开发布仍需00任务、许可与伦理复核。

| 资产 | Git | 可公开 | 条件/原因 |
|---|---:|---:|---|
| 处理/验证脚本 | 是 | 可考虑 | 不含密钥、私密路径或受限内容 |
| canonical schema与小型manifest | 是 | 可考虑 | 仅含统计、hash、工作ID与状态 |
| Data Card/Datasheet/审计报告 | 是 | 可考虑 | 保留限制与未知项，不嵌入敏感样本 |
| CSMV视频级派生JSONL | 否 | 当前否 | 先完成许可、第二主集与发布复核 |
| CSMV评论正文、评论ID、用户标识 | 否 | 否 | 标签来源敏感、T0泄漏、隐私风险 |
| TikTok/Facebook/新闻媒体、截图、URL清单 | 否 | 否 | 第三方版权、平台条款与可撤回性 |
| CUC原始文件、48维向量、canonical | 否 | 否 | 许可未知、本地限定、含非T0遗留特征 |
| 模型权重、索引、embedding | 否 | 当前不存在 | G1/G2前禁止生成正式工件 |
| 50—100条人工错误审查内容 | 否 | 否 | 仅数据缺陷发现，不是正式优越性证据 |

## 发布前硬门

1. G1确认两个合法可用公开人工主集且第二主集冻结。
2. 两个主集均完成不可变manifest、标签映射和版本化split。
3. 正式split运行全部Critical泄漏测试零失败，且语义近重复/同源事件边界经人工复核。
4. G2确认100% lineage、标签物理隔离、test评论物理隔离和manifest重跑。
5. 00任务书面批准发布范围；未经批准不得把本地候选改名为正式benchmark。
